# 🚀 Git: Temporarily Switch Branches Without Losing Changes (Ubuntu 25.04)

Best & safest method for 99% of cases → `git stash`.  
Cleaner than temp commits, zero history pollution, perfect for CI/CD runners too.

## 📌 Method 1 – git stash (Recommended)

```bash
# 1. On feature-x, save everything (including untracked files)
git stash push -u -m "wip: temporary switch to develop"

# 2. Switch freely
git switch develop          # or: git checkout develop

# 3. Do whatever you need on develop (pull, fix hotfix, etc.)

# 4. Return to your branch
git switch feature-x

# 5. Restore changes (pop = apply + drop)
git stash pop               # latest stash
# or specific one → git stash pop stash@{2}
```

### Why `pop` > `apply`?
- `pop` removes the stash automatically on success → cleaner stack.
- If conflicts → stash stays safe, you resolve, then `git stash drop`.

### Useful stash commands you’ll actually use daily
```bash
git stash list              # see all stashes
git stash show -p stash@{0} # preview a stash
git stash branch new-branch stash@{1}  # create branch from old stash
git stash clear             # nuke everything (careful
```

## 📌 Method 2 – git worktree (God-tier for parallel work)

When you work on two branches at the same time a lot (very common in Go microservices).

```bash
# Create separate folder for develop, completely independent
git worktree add ../myproject-develop develop

# Work normally in ../myproject-develop
cd ../myproject-develop
# ... make commits, run tests, etc.

# When finished
cd ../myproject              # back to original
git worktree remove ../myproject-develop
```

Zero stashing, zero conflicts, zero context switching pain.  
My personal default workflow since Git 2.23.

## 🎯 Fixed & Clean Demo Script (run it!)

```bash
#!/usr/bin/env bash
set -e

rm -rf /tmp/git-test && mkdir /tmp/git-test && cd /tmp/git-test

git init -b main
echo "initial" > file.txt
git add . && git commit -m "init"

git switch -c feature-x
echo "work in progress" >> file.txt
echo "*.log" > .gitignore          # untracked file example
touch temp.log                     # untracked

echo "=== Before stash ==="
git status --no-pager status --short

git stash push -u -m "wip before hotfix"

echo "=== After stash (clean) ==="
git status --short

git switch main
echo "hotfix" > hotfix.txt
git add . && git commit -m "quick hotfix"

git switch feature-x
git stash pop

echo "=== Changes restored ==="
git status --short
cat file.txt
```

## 💡 Pro Tips for Go Backend Devs

- In GitHub Actions / GitLab CI on Ubuntu runners → always stash or use `--work-tree` when checking out multiple branches in same job.
- Combine with `git switch --guess` or `git switch --detach` for even faster navigation.
- Never use temporary commits on feature branches → they make interactive rebase painful later.
- If you forget what you stashed → `git stash show -p` is your best friend.
