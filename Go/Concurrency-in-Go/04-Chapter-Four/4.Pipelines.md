## âš¡ Introducing Channels for Concurrent Pipelines

To leverage Goâ€™s concurrency and improve pipeline scalability, we use **channels** to create stream-oriented pipelines where stages run concurrently.

### ğŸ“Œ Channel-Based Pipeline Stage

A channel-based stage:
- Receives data from an input channel.
- Processes it.
- Sends results to an output channel.

Hereâ€™s an example converting the `multiply` stage to use channels:

```go

    out := make(chan int)

        defer close(out)

        }
    }()
    return out
}
```

**How It Works**:
- **Input**: Reads from `in` channel.
- **Processing**: Multiplies each value by `multiplier`.
- **Output**: Sends results to `out` channel.
- **Concurrency**: Runs in a goroutine, closing the output channel when done.

Similarly, for the `add` stage:


## ğŸ“Š Batch vs. Stream Processing

| **Aspect**             | **Batch Processing**                     | **Stream Processing**                    |
|------------------------|------------------------------------------|------------------------------------------|

**Tip** ğŸ’¡: Use batch processing for small, predictable datasets and stream processing with channels for large or real-time data.
