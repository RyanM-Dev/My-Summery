## ğŸ“Œ What is a Filter in MongoDB?

A **filter** is a condition you pass to MongoDB to tell it:

## ğŸ”° 1. When a Filter **IS** Required

âœ… Use filters when targeting **specific documents** to **find**, **update**, or **delete**.

### Examples:


> âš ï¸ Without a filter, commands may affect **all documents**!

## âŒ 2. When a Filter is **NOT** Required

You can **skip** the filter:

- When inserting new documents

- When counting all documents
    

### Examples:

```js

```

---

### ğŸ”¸ A. Simple Equality

### ğŸ”¸ B. Comparison Operators

|Operator|Meaning|Example|
|---|---|---|
|`$gt`|Greater than|`{ age: { $gt: 25 } }`|
|`$lt`|Less than|`{ age: { $lt: 40 } }`|

|`$in`|In a list|`{ name: { $in: ["Ali", "Sara"] } }`|
|`$nin`|Not in a list|`{ age: { $nin: [20, 21, 22] } }`|

---

### ğŸ”¸ C. Logical Operators

|Operator|Use|
|---|---|
|`$and`|All conditions must be true|
|`$or`|At least one condition is true|
|`$not`|Negate a condition|
|`$nor`|None of the conditions must be true|

#### Example:

```js

    { age: { $lt: 18 } },
```

### ğŸ”¸ D. Field Existence & Type

### ğŸ”¸ E. Array Filters
